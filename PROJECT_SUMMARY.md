## 🎯 核心功能实现

### 1. 音频处理流程
```
上传 → 验证(300MB) → 保存 → 转写 → 说话人识别 → 完成
```
- 支持6种音频格式（MP3, M4A, WAV, WEBM, MP4, MPEG）
- 前端实时进度显示
- 自动生成初始标题（时间戳格式）

### 2. LLM集成
- **默认服务**: 豆包（火山引擎）
- **备用服务**: 千问、Claude、OpenAI、智谱GLM
- **功能**: 音频转文字、说话人识别、纪要生成、标题生成
- **配置**: Web界面配置、实时验证

### 3. 会议纪要生成
- **预设模板**: 3个专业模板（客户需求、技术讨论、通用）
- **自定义模板**: 用户可保存个人模板
- **多版本管理**: 支持平行版本对比
- **导出功能**: Markdown、TXT格式

### 4. 数据管理
- **音频清理**: 7天自动删除 + 手动删除
- **存储策略**:
  - 音频文件：7天后自动删除
  - 转写文本：永久保存
  - 会议纪要：永久保存（所有版本）
  - 说话人映射：永久保存

### 5. 前端界面
- **主页**: 音频上传、最近会议、存储状态提醒
- **详情页**: 转写结果、说话人管理、纪要生成、版本时间线
- **历史页**: 搜索过滤、状态筛选、时间线展示
- **设置页**: LLM配置、API验证、服务切换

## 🚀 部署指南

### 1. 环境要求
- Python >= 3.12
- pip（Python包管理器）

### 2. 安装步骤
```bash
# 克隆项目（如果需要）
git clone <repository-url>
cd my-meeting-recording

# 安装依赖
pip install -e .

# 安装可选依赖（用于定时清理和增强LLM功能）
pip install aiohttp schedule

# 运行数据库迁移
python scripts/migrate.py

# 初始化预设模板
python scripts/init_templates.py
```

### 3. 配置AI服务
```bash
# 复制配置模板
cp config.example.json config.json

# 编辑配置文件，填入你的豆包API密钥
# 获取API密钥: https://console.volcengine.com/ark
```

### 4. 启动服务
```bash
# 方式1：使用Makefile
make web

# 方式2：直接运行
python main.py
```

### 5. 访问应用
```
主页: http://localhost:8000/static/index.html
历史: http://localhost:8000/static/history.html
设置: http://localhost:8000/static/settings.html
```
